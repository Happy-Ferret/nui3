 What is NGL ?

NGL is an application framework intended to work with SGI's OpenGL* API. It is similar in many ways to glut (the GL utility toolkit), but with many enhancements in order to make it an easy and yet powerful tool for cross-platform C++ application programming.

It is currently developped on Windows (98, Me, NT/XP/2000), Linux (2.x / x86), FreeBSD (5.0 / x86) and MacOS (9.x and X).

Features :
  • C++ portable framework
  • builtin interactive console and flexible logging facilities
  • window'ed / fullscreen mode, video mode switching
  • various inputs (keyboard, mouse, joystick)
  • seamless usage of OpenGL extensions
  • extensible image loader/exporter (PNG, JPG, PPM and TGA)
  • advanced font support (via FreeType, many font formats - eg. TrueType)
  • mesh loader (3DS)
  • internationalisation support (Unicode and SBCS)
  • comprehensive documentation
While NGL is quite generic and designed for all purposes, you might have a look at other related projects which could better suits your needs.

* OpenGL is a trademark of Silicon Graphics Incorporated.

 What is NUI ?

NUI is a cross-platform C++ GUI programming framework for OpenGL applications built on top of the NGL framework. In other words, its purpose is to help programmers write multi-platform GUI applications with a simple knowledge in C++. The rendering being based on OpenGL, graphics can intensively use alpha blending, anti-aliasing, complex primitives and even 3D with full hardware acceleration when available.

Features :
  • modern and versatile layout (à la GTK+)
  • sophisticated widgets (text control, tree view, 3D mesh viewer, SVG graphics)
  • XML based UI description, can load or serialize UI parts easily
  • theme support
  • a RADtool project is already on the way
Have a look at the screenshots section for some examples.

Thanks to Activisu Solutions who sponsored NUI development.

 How do I use it ?

You will find anything you need on this site :
  • download a binary release for your platform or fetch sources and compile the library and examples yourself. While the build process is easy and need a few minutes, you are highly encouraged to look first at the README file included in every source release.
  • Have a look at the examples code and keep the API reference documentation at hand. The documentation is included with every binary and source release for offline browsing.
If you want to go further (submit a bug, participate in code/doc, propose a new platform), or just like to be on the bleeding edge, you should use the CVS repository. Of course, you are highly encouraged to go there and send us patches, feedback or any good ideas !

Available mailing lists :
  • ngl-devel : subscribe | archive
    If you want to help in development or need yourself some help, subscribe here.

  • ngl-cvs : subscribe | archive
    CVS commit automatic reports as generated by CVSreport.
You can also try the #ngl channel on irc.freenode.net, the authors are often hanging around here.
